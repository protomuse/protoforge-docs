# Searching the ProtoForge Track Catalogue

<Callout type="info" title="What This Guide Covers">
  How to search and filter the ProtoForge track catalogue using query parameters, paginate results, and interpret track metadata including rights holders.
</Callout>

## Prerequisites

- Authenticated with the ProtoForge API ([Authentication Guide](/guides/authentication))

<Steps>

### Basic Search

The simplest search uses the `q` parameter for full-text search across track titles, artists, and descriptions:

<CodeGroup>
```typescript TypeScript
const response = await fetch(
  "https://api.protoforge.com/v1/tracks/search?q=cinematic",
  {
    headers: {
      "Authorization": "Bearer pk_test_your_api_key",
      "Content-Type": "application/json"
    }
  }
);

const data = await response.json();
console.log(`Found ${data.pagination.total} tracks`);
```

```bash cURL
curl -X GET "https://api.protoforge.com/v1/tracks/search?q=cinematic" \
  -H "Authorization: Bearer pk_test_your_api_key"
```

```python Python
import requests

response = requests.get(
    "https://api.protoforge.com/v1/tracks/search",
    params={"q": "cinematic"},
    headers={"Authorization": "Bearer pk_test_your_api_key"}
)

data = response.json()
print(f"Found {data['pagination']['total']} tracks")
```
</CodeGroup>

**Response:**

```json
{
  "data": [
    {
      "id": "trk_def456",
      "title": "Cinematic Horizons",
      "artist": "Orchestral Works",
      "duration_seconds": 185,
      "bpm": 90,
      "key": "D major",
      "genre": "cinematic",
      "mood": ["dramatic", "inspiring"],
      "isrc": "GB-XYZ-24-00042",
      "rights_holders": [
        { "name": "Sarah Chen", "role": "writer", "share_percentage": 60 },
        { "name": "Horizon Publishing", "role": "publisher", "share_percentage": 40 }
      ],
      "available_territories": ["GB", "US", "DE", "FR", "JP"],
      "status": "available"
    }
  ],
  "pagination": { "page": 1, "per_page": 20, "total": 23 }
}
```

### Filter by Musical Properties

Combine query parameters to narrow results by BPM range, musical key, genre, and mood:

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `q` | string | Full-text search | `cinematic` |
| `bpm_min` | integer | Minimum BPM | `100` |
| `bpm_max` | integer | Maximum BPM | `140` |
| `key` | string | Musical key | `A minor` |
| `genre` | string | Genre filter | `electronic` |
| `mood` | string | Mood filter | `energetic` |
| `territory` | string | ISO 3166-1 alpha-2 | `GB` |
| `status` | enum | `available`, `licensed`, `restricted` | `available` |

<CodeGroup>
```typescript TypeScript
const params = new URLSearchParams({
  genre: "electronic",
  bpm_min: "120",
  bpm_max: "140",
  key: "A minor",
  territory: "GB",
  status: "available"
});

const response = await fetch(
  `https://api.protoforge.com/v1/tracks/search?${params}`,
  {
    headers: { "Authorization": "Bearer pk_test_your_api_key" }
  }
);

const data = await response.json();
```

```bash cURL
curl -X GET "https://api.protoforge.com/v1/tracks/search?genre=electronic&bpm_min=120&bpm_max=140&key=A+minor&territory=GB&status=available" \
  -H "Authorization: Bearer pk_test_your_api_key"
```

```python Python
import requests

response = requests.get(
    "https://api.protoforge.com/v1/tracks/search",
    params={
        "genre": "electronic",
        "bpm_min": 120,
        "bpm_max": 140,
        "key": "A minor",
        "territory": "GB",
        "status": "available"
    },
    headers={"Authorization": "Bearer pk_test_your_api_key"}
)
```
</CodeGroup>

**Response:**

```json
{
  "data": [
    {
      "id": "trk_abc123",
      "title": "Neon Pulse",
      "artist": "Synthwave Collective",
      "bpm": 128,
      "key": "A minor",

      "mood": ["energetic", "uplifting"],
      "status": "available"
    }
  ],
  "pagination": { "page": 1, "per_page": 20, "total": 3 }
```

### Paginate and Sort Results

Use `page`, `per_page`, and `sort` to navigate large result sets:

<CodeGroup>
```typescript TypeScript
const response = await fetch(
  "https://api.protoforge.com/v1/tracks/search?genre=pop&page=2&per_page=10&sort=bpm",
  {
    headers: { "Authorization": "Bearer pk_test_your_api_key" }
  }
);

const data = await response.json();
console.log(`Page ${data.pagination.page} of ${Math.ceil(data.pagination.total / 10)}`);
```

```bash cURL
curl -X GET "https://api.protoforge.com/v1/tracks/search?genre=pop&page=2&per_page=10&sort=bpm" \
  -H "Authorization: Bearer pk_test_your_api_key"
```

```python Python
import requests
import math

response = requests.get(
    "https://api.protoforge.com/v1/tracks/search",
    params={"genre": "pop", "page": 2, "per_page": 10, "sort": "bpm"},
    headers={"Authorization": "Bearer pk_test_your_api_key"}
)

data = response.json()
total_pages = math.ceil(data["pagination"]["total"] / 10)
print(f"Page {data['pagination']['page']} of {total_pages}")
```
</CodeGroup>

**Sort options:** `relevance` (default), `title`, `bpm`, `duration`

## Understanding Track Metadata

### Rights Holders

Every track includes a `rights_holders` array with the name, role, and revenue share percentage for each party. Shares always sum to exactly 100%.

| Role | Description |
|------|-------------|
| `writer` | Songwriter or composer |
| `publisher` | Music publisher controlling the composition |
| `performer` | Recording artist or performer |

### Track Status

| Status | Meaning | Can License? |
|--------|---------|--------------|
| `available` | Track is available for licensing | Yes |
| `licensed` | Track has an active exclusive license | No (until license expires) |
| `restricted` | Rights metadata incomplete | No |

### ISRC Codes

The `isrc` field contains the International Standard Recording Code in the format `CC-XXX-YY-NNNNN`:

- `CC` — Country code (ISO 3166-1 alpha-2)
- `XXX` — Registrant code
- `YY` — Year of registration
- `NNNNN` — Designation code

<Accordion title="Common Mistakes">

**Not paginating results**
The default `per_page` is 20. For large catalogues, always check `pagination.total` and iterate through pages.

**Invalid territory codes**
Territory codes must be ISO 3166-1 alpha-2 (e.g., `GB` not `UK`, `US` not `USA`).

**Filtering by multiple moods**
The `mood` parameter accepts a single value. To search multiple moods, make separate requests or use the `q` parameter.

</Accordion>
